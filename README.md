# VanillaAuc 🛒

Легковесный, оптимизированный и отзывчивый плагин на асинхронный аукцион для серверов Minecraft (Paper/Purpur 1.21+), написанный с использованием современного **Brigadier API** и **MiniMessage**.

Основная фишка плагина — экономика, завязанная на **алмазной руде (АР)** в качестве физической валюты, а также полностью динамический интерфейс с обновлением сроков действия лотов в реальном времени.

---

## ✨ Особенности (Features)

*   **Физическая валюта:** Никаких виртуальных балансов и Vault. Торговля ведется за алмазную руду (`DIAMOND_ORE` и `DEEPSLATE_DIAMOND_ORE`). При покупке плагин автоматически списывает руду из инвентаря покупателя.
*   **Асинхронная архитектура (Async SQL):** Все операции с базой данных SQLite (загрузка лотов, покупка, логирование, проверка лимитов) вынесены в асинхронные потоки (`BukkitRunnable`). Сервер не будет фризить и лагать даже при активной торговле.
*   **Real-time обновление:** Сроки годности товаров (3 суток) тикают в реальном времени каждую секунду прямо внутри GUI без раздражающего переоткрытия меню или дерганья курсора.
*   **Защита от дюпов и Race Conditions:** База данных строго контролирует статус лота. Если два игрока одновременно попытаются купить один предмет, плагин успешно обработает запрос первого, а второму выдаст мягкое предупреждение без потери ресурсов.
*   **Современный UI/UX:** Кастомные градиенты и стили текста через компоненты `Adventure MiniMessage`. Полная валидация кликов за пределами инвентаря (без крашей в консоль).

---

## 🛠 Команды и Права

Плагин использует современный движок команд Mojang Brigadier:

*   `/ah` — Открыть главное меню аукциона.
*   `/ah sell <цена>` — Выставить предмет из главной руки на продажу.
    *   *Защита:* Максимальная цена ограничена вместимостью инвентаря покупателя (2304 АР), а максимальное количество активных слотов на одного игрока — **5 лотов**.

---

## 🏗 Архитектура проекта

Код разбит на независимые слои, что позволяет легко его поддерживать:
*   `AuctionItem` — неизменяемая запись (Java Record) для представления структуры лота.
*   `DatabaseManager` — полностью изолированный слой для работы с SQLite.
*   `GUI / Holders` — разделение логики интерфейса и его содержимого через кастомные `InventoryHolder`.
*   `Listeners` — модульные слушатели событий (`InventoryClickEvent`) с проверками на безопасность транзакций.

## ⚙️ Установка и Требования

1. Соберите проект через Gradle.
2. Поместите плагин в папку `plugins` вашего сервера.
3. Версия ядра: **Paper 1.21** или новее (требуется поддержка библиотек Paper Brigadier и Adventure).
4. Запустите сервер. Плагин автоматически создаст базу данных `auction.db` и все необходимые таблицы.
